1.4 Brand Vetting Criteria  ✅ Locked

Every brand applying to BOMY Consortium must meet all five criteria:

#Criterion
1Brand must be first registered in Malaysia (SSM registration required)
2Product must be at least 50% made in Malaysia
3Must be owned by at least 1 Malaysian citizen or Permanent Resident
4BOMY deals only with the owner, founder, co-founder, family inheritor, or authorised person in charge of the brand — no agents, distributors, or resellers
5No multi-level marketing (MLM) structure allowed on the platform
Approval process: Final approval requires unanimous agreement from 4 designated stakeholders. Charlie to confirm the 4 roles (e.g. Charlie, Bob, Ops Lead, Finance Lead). A single veto blocks approval — no brand is listed without all 4 approvals. This is enforced in the admin workflow, not left to convention.

Appendix A

i18n Architecture Note

The language roadmap (EN → BM → ZH → Others) requires this pattern from day one:

apps/web/
└── messages/
    ├── en.json          ← Launch
    ├── bm.json          ← V1 post-launch
    ├── zh.json          ← V2
    └── [locale].json    ← V3+

Every user-facing string is a key, never hardcoded:

// ✅ Correct
const t = useTranslations('checkout')
return <h1>{t('title')}</h1>

// ❌ Wrong — hardcoded, untranslatable
return <h1>Checkout</h1>

Namespace structure mirrors site IA: common, checkout, account, seller, admin, membership.

Appendix C

Brand Vetting Database Design (Stage 2 Note)

// packages/db/schema/brandVetting.ts
export const brandVettingApplications = pgTable('brand_vetting_applications', {
  id: uuid('id').primaryKey().defaultRandom(),
  brandId: uuid('brand_id').references(() => brands.id),
  ssmRegistrationNumber: text('ssm_registration_number').notNull(),
  madeInMalaysiaPercentage: integer('made_in_malaysia_pct').notNull(), // >= 50
  ownerIsMalaysian: boolean('owner_is_malaysian').notNull(),
  contactRole: contactRoleEnum('contact_role').notNull(),
  noMlmDeclaration: boolean('no_mlm_declaration').notNull(),
  submittedAt: timestamp('submitted_at').notNull().defaultNow(),
  // 4 stakeholder votes (null = not yet voted)
  vote1StakeholderId: uuid('vote1_stakeholder_id').references(() => users.id),
  vote1Decision: voteDecisionEnum('vote1_decision'), // approve/reject/abstain
  vote1At: timestamp('vote1_at'),
  // ... vote2, vote3, vote4 same pattern ...
  finalStatus: vettingStatusEnum('final_status').default('pending'),
  finalAt: timestamp('final_at'),
})
// Approved only when all 4 votes = 'approve'
